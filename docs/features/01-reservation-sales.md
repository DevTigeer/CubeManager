# Feature 01: 예약/매출 관리

## 1. 개요

cubeescape.co.kr의 예약 테이블을 연동하여 일별 예약 현황을 조회하고,
결제 수단별(카드/현금/계좌이체) 매출을 관리하는 기능.

---

## 2. 웹 연동 상세

### 2.1 로그인 프로세스

1. `POST http://www.cubeescape.co.kr/bbs/login.php` 로 로그인
   - Form Data: `mb_id=<설정에서 입력>`, `mb_password=<설정에서 입력>`
   - 응답 Cookie(세션) 저장
2. 세션 유지하며 예약 테이블 조회

### 2.2 예약 데이터 조회

- **URL 패턴**: `http://www.cubeescape.co.kr/adm/room_list.php?sfl=r_date&stx={YY-MM-DD}`
  - 예: `stx=26-03-18` → 2026년 3월 18일
- **파싱 대상**: HTML 테이블 (`<table>` 태그)
- **추출 필드**: 시간대, 방 이름, 예약자, 연락처, 인원, 상태

### 2.3 동기화 정책

| 항목 | 설명 |
|------|------|
| 수동 조회 | 날짜 입력 후 조회 버튼 클릭 |
| 자동 갱신 | 10분 간격 (현재 조회 중인 날짜) |
| 캐시 | DB에 저장, 오프라인 시 캐시 데이터 표시 |

## 4. 매출 관리

### 4.1 결제 수단 분류

| 결제 수단 | 색상 태그 | 아이콘 |
|-----------|----------|--------|
| 카드 결제 | 🔵 파란색 | 💳 |
| 현금 결제 | 🟢 초록색 | 💵 |
| 계좌 이체 | 🟡 노란색 | 🏦 |

### 4.2 매출 입력 UI

```
┌──────────────────────────────────────────────┐
│           2026-03-18 매출 현황                │
├──────────────────────────────────────────────┤
│  [+ 매출 추가]  [+ 지출 추가]                │
├──────┬────────┬──────────┬────────┬─────────┤
│  #   │ 항목   │ 금액     │ 결제   │ 구분    │
├──────┼────────┼──────────┼────────┼─────────┤
│  1   │ 방1-홍 │ 25,000   │ 🔵카드 │ 매출   │
│  2   │ 방2-이 │ 30,000   │ 🟢현금 │ 매출   │
│  3   │ 비품   │ -5,000   │ 🟢현금 │ 지출   │
├──────┴────────┼──────────┴────────┴─────────┤
│               │                              │
│  카드 합계    │              25,000원         │
│  현금 합계    │              25,000원         │
│  계좌 합계    │                   0원         │
│  ─────────────┼─────────────────────────     │
│  총 매출      │              50,000원         │
│  총 지출      │              -5,000원         │
│  순 매출      │              45,000원         │
└───────────────┴──────────────────────────────┘
```

### 4.3 현금 잔액 관리

```
현금 잔액 = 전일 마감 현금 잔액
           + 당일 현금 수입 (현금 결제 매출)
           - 당일 현금 지출 (비품 구매, 현금 출금 등)
```

- 전일 마감 잔액은 `cash_balance` 테이블에서 자동 조회
- 당일 현금 거래가 발생할 때마다 실시간 업데이트
- 마감 시 자동으로 다음 날 개시 잔액에 이월

## 5. 비즈니스 로직

### 5.1 ReservationService

```
- Login(id, pw) → 세션 쿠키 반환
- FetchReservations(date) → List<Reservation>
- ParseHtmlTable(html) → List<Reservation>
- SyncReservations(date) → DB 저장
```

### 5.2 SalesService

```
- GetDailySales(date) → DailySales
- AddSaleItem(item) → 매출/지출 항목 추가
- UpdateSaleItem(item) → 항목 수정
- DeleteSaleItem(id) → 항목 삭제
- CalculateTotals(date) → 결제수단별 합계
- GetCashBalance(date) → 현금 잔액 조회
- CloseDailyBalance(date) → 일 마감 처리
```

## 6. 예외 처리

| 상황 | 처리 |
|------|------|
| 웹 연결 실패 | 캐시 데이터 표시 + "오프라인" 표시 |
| 로그인 실패 | 에러 메시지 + 재로그인 안내 |
| 세션 만료 | 자동 재로그인 시도 |
| 파싱 실패 | 로그 기록 + 수동 입력 안내 |
