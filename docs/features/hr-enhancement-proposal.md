## 1. 자동 알림 시스템

> **핵심 원칙**: 문제가 발생하면 관리자가 찾아보기 전에 시스템이 먼저 알린다.

### 1-1. 체크리스트 미완료 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 출근 후 1시간 이내 체크리스트 0% 시 |
| **알림 방식** | 화면 상단 경고 배너 + 기록 저장 |
| **기록 내용** | 날짜, 직원명, 미완료 항목 수, 경과시간 |
| **관리자 조회** | 관리자탭 > 알림이력 > 체크리스트 미이행 |

**동작 흐름:**
```
출근 기록 → 1시간 타이머 시작 → 만료 시 체크리스트 확인
  → 0% → 경고 팝업 + alert_logs 테이블 기록
  → 50% 미만 → 경고 배너 (팝업 없음) + 기록
  → 50% 이상 → 정상 (기록 없음)
```

### 1-2. 지각 누적 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 월간 지각 3회 이상 |
| **알림 방식** | 관리자 대시보드 경고 카드 |
| **기록 내용** | 직원명, 월간 지각횟수, 누적패턴 |
| **액션** | 경고 발급 가능 (1차/2차/3차 경고) |

### 1-3. 인수인계 미확인 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 출근 후 30분 이내 미확인 인수인계 존재 시 |
| **알림 방식** | 하늘색 팝업 "읽지 않은 인수인계가 있습니다" |
| **기록 내용** | 인수인계ID, 대상 직원, 경과시간 |

### 1-4. 현금 잔액 이상 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 일 마감 시 현금잔액 차이 ≥ ±5,000원 |
| **알림 방식** | 마감 체크리스트 완료 시 자동 검증 |
| **기록 내용** | 날짜, 예상잔액, 실제잔액, 차이, 담당자 |

### 1-5. 재고 부족 알림

| 항목 | 내용 |
|------|------|
| **트리거** | 현재수량 < 필요수량의 50% |
| **알림 방식** | 사이드바 📦 아이콘에 빨간 뱃지 |
| **기록 내용** | 품목명, 부족수량, 마지막 보충일 |

## 2. 근무 태도 추적 & 점수화

> **핵심 원칙**: 객관적 데이터로 근무 태도를 수치화하여 공정한 평가 기반 마련.

### 2-1. 근무 점수 시스템 (100점 만점/월)

| 항목 | 배점 | 감점 기준 |
|------|:---:|----------|
| **출퇴근 준수** | 40점 | 지각 1회 -3점, 조퇴 1회 -2점, 무단결근 -10점 |
| **체크리스트 이행** | 25점 | 미완료 항목 1개당 -1점, 전체 미이행 -5점/일 |
| **인수인계 확인** | 15점 | 미확인 1건당 -3점, 30분 초과 확인 -1점 |
| **현금 정산 정확도** | 10점 | ±5,000원 이상 차이 1건당 -2점 |
| **마감 체크리스트** | 10점 | 마감 항목 미완료 1개당 -2점 |

**DB 테이블: `employee_scores`**
```sql
CREATE TABLE employee_scores (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    score_month TEXT NOT NULL,        -- 'YYYY-MM'
    attendance_score INTEGER DEFAULT 40,
    checklist_score INTEGER DEFAULT 25,
    handover_score INTEGER DEFAULT 15,
    cash_accuracy_score INTEGER DEFAULT 10,
    closing_score INTEGER DEFAULT 10,
    total_score INTEGER DEFAULT 100,
    FOREIGN KEY (employee_id) REFERENCES employees(id)
);
```

### 2-2. 알림 이력 테이블

**DB 테이블: `alert_logs`**
```sql
CREATE TABLE alert_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    alert_type TEXT NOT NULL,         -- 'checklist_delay', 'late_arrival',
                                     -- 'handover_unread', 'cash_discrepancy',
                                     -- 'inventory_shortage'
    employee_id INTEGER,             -- NULL이면 시스템 알림
    alert_date TEXT NOT NULL,        -- 'YYYY-MM-DD'
    alert_time TEXT NOT NULL,        -- 'HH:MM:SS'
    severity TEXT DEFAULT 'warning', -- 'info', 'warning', 'critical'
    message TEXT NOT NULL,
    is_resolved INTEGER DEFAULT 0,   -- 0=미해결, 1=해결
    resolved_by TEXT,
    resolved_at TEXT,
    FOREIGN KEY (employee_id) REFERENCES employees(id)
);
```

### 2-3. 경고 이력 테이블

**DB 테이블: `warnings`**
```sql
CREATE TABLE warnings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    warning_level INTEGER NOT NULL,   -- 1=구두경고, 2=서면경고, 3=최종경고
    reason TEXT NOT NULL,             -- '월간 지각 5회 (3월)'
    issued_date TEXT NOT NULL,
    issued_by TEXT NOT NULL,          -- 관리자명
    acknowledged INTEGER DEFAULT 0,   -- 직원 확인 여부
    acknowledged_at TEXT,
    note TEXT,
    FOREIGN KEY (employee_id) REFERENCES employees(id)
);
```

## 3. 체크리스트 강화

### 3-1. 시간 제한 체크리스트

| 기능 | 설명 |
|------|------|
| **마감시한** | 각 항목에 "출근 후 N분 이내" 시한 설정 가능 |
| **지연 표시** | 시한 초과 항목은 주황색 배경 + ⚠️ 아이콘 |
| **알림 연동** | 시한 초과 시 alert_logs에 자동 기록 |

**템플릿 확장 필드:**
```
deadline_minutes INTEGER DEFAULT 0  -- 0이면 제한 없음, 60이면 출근 후 60분
```

### 3-2. 사진 첨부 체크리스트

| 기능 | 설명 |
|------|------|
| **증거 첨부** | 청소, 정리 등 완료 사진 첨부 가능 |
| **저장 경로** | `%APPDATA%/CubeManager/checklist-photos/YYYY-MM-DD/` |
| **관리자 열람** | 관리자탭에서 날짜별 첨부 사진 열람 |

### 3-3. 체크리스트 완료율 통계

| 기간 | 표시 내용 |
|------|----------|
| **일별** | 오늘 진행률 바 (현재 존재) |
| **주별** | 주간 평균 완료율 + 미완료 항목 Top 3 |
| **월별** | 직원별 월간 체크리스트 이행률 그래프 |
| **관리자 조회** | 관리자탭 > 체크리스트 통계 하위탭 |

## 4. 출퇴근 확장

### 4-1. 무단결근 자동 감지

```
매일 12:00에 자동 체크:
  스케줄이 있는데 출근 기록이 없는 직원 → '무단결근' 상태 기록
  → alert_logs에 'no_show_employee' 기록
  → 관리자 대시보드에 즉시 표시
```

### 4-2. 초과근무 자동 감지

| 기능 | 설명 |
|------|------|
| **감지 기준** | 실제 퇴근시간 - 스케줄 퇴근시간 > 30분 |
| **기록** | overtime_minutes 필드 추가 |
| **급여 연동** | 초과근무 시간 × 시급 × 1.5 자동 계산 (선택적) |

### 4-3. 출퇴근 패턴 분석

**관리자 조회 화면:**
```
┌────────────────────────────────────────────┐
│  김민수 - 3월 출퇴근 패턴                    │
├────────────────────────────────────────────┤
│  총 근무일: 22일                             │
│  정상 출근: 18일 (82%)                       │
│  지각: 3일 (14%) ← 평균 12분 지각            │
│  조퇴: 1일 (5%)                             │
│  무단결근: 0일                               │
│                                            │
│  지각 요일 패턴: 월요일 2회, 수요일 1회       │
│  평균 출근시간: 09:57 (스케줄 10:00)          │
│  평균 퇴근시간: 17:03 (스케줄 17:00)          │
│                                            │
│  [경고 발급]  [상세 이력]                    │
└────────────────────────────────────────────┘
```

### 4-4. 결근 사유 관리

**DB 테이블: `absence_records`**
```sql
CREATE TABLE absence_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    employee_id INTEGER NOT NULL,
    absence_date TEXT NOT NULL,
    absence_type TEXT NOT NULL,        -- 'sick', 'vacation', 'personal', 'unauthorized'
    reason TEXT,
    is_approved INTEGER DEFAULT 0,
    approved_by TEXT,
    approved_at TEXT,
    FOREIGN KEY (employee_id) REFERENCES employees(id)
);
```

|------|------|

## 6. 스케줄 지능화

### 6-1. 스케줄 템플릿

| 기능 | 설명 |
|------|------|
| **템플릿 저장** | "기본 주간패턴" 저장 (직원+요일+시간 조합) |
| **한번에 적용** | 월 선택 → [템플릿 적용] → 전체 주차 자동 생성 |
| **예외 편집** | 적용 후 개별 일자만 수정 가능 |

### 6-2. 충돌 감지

```
스케줄 추가 시 자동 검증:
  ① 같은 직원 같은 날 중복 스케줄 → 경고
  ② 전날 마감(01:00) → 다음날 오픈(10:00) = 9시간 간격 미만 → 경고
  ③ 주간 근무시간 52시간 초과 → 근로기준법 경고
```

### 6-3. 최소 인원 보장

| 기능 | 설명 |
|------|------|
| **설정** | 요일/시간대별 최소 필요 인원 설정 |
| **검증** | 스케줄 작성 후 미달 시간대 빨간 표시 |
| **예시** | 금요일 18:00~22:00 최소 3명 → 2명만 배치 시 경고 |

### 6-4. 대타 요청 시스템

| 기능 | 설명 |
|------|------|
| **요청** | 직원이 특정 날짜 "대타 요청" 등록 |
| **매칭** | 해당 날짜 비근무 직원 목록 표시 |
| **확정** | 관리자가 대타 승인 → 스케줄 자동 교체 |

## 7. 관리자 대시보드 강화

### 7-1. 실시간 운영 현황

```
┌─────────────────────────────────────────────────┐
│  📊 실시간 운영 현황               2026-03-23    │
├──────────┬──────────┬──────────┬────────────────┤
│ 현재 근무자 │ 오늘 매출  │ 체크리스트 │  알림          │
│   3 / 3명  │ ₩456,000  │  8/12(67%)│  ⚠️ 2건       │
│   ✅ 정상   │ 목표대비83%│  지연 1건  │  미확인 1건    │
├──────────┴──────────┴──────────┴────────────────┤
│  ⚠️ 알림 목록                                    │
│  • 10:32 김민수 체크리스트 미완료 (출근 후 62분)    │
│  • 10:05 인수인계 #47 미확인 (박서연 미열람)        │
└─────────────────────────────────────────────────┘
```

### 7-2. 직원별 월간 리포트

| 항목 | 표시 내용 |
|------|----------|
| **근무 점수** | 100점 만점 원형 차트 |
| **출퇴근** | 정상/지각/조퇴/결근 비율 바 차트 |
| **체크리스트** | 일별 완료율 추이선 |
| **경고 이력** | 1차/2차/3차 경고 타임라인 |
| **급여 요약** | 기본급 + 수당 + 공제 합계 |

### 7-3. 알림 이력 조회

| 필터 | 옵션 |
|------|------|
| **기간** | 시작일 ~ 종료일 |
| **유형** | 체크리스트/지각/인수인계/현금/재고 |
| **심각도** | 정보/경고/심각 |
| **직원** | 전체 또는 특정 직원 |
| **상태** | 해결/미해결 |

### 7-4. 월간 운영 보고서

```
자동 생성 항목:
  ① 매출 추이 (일별 매출 테이블)
  ② 인원 효율 (매출 ÷ 투입시간)
  ③ 직원별 근무시간 합산
  ④ 체크리스트 전체 이행률
  ⑤ 알림 발생 건수 (유형별)
  ⑥ 현금 정산 정확도
  ⑦ 경고 발급 현황
```

## 8. 급여 연동 자동화

### 8-1. 감점 자동 반영

| 감점 사유 | 감점액 | 근거 |
|----------|:------:|------|
| 무단결근 | 해당일 급여 전액 | 근무 미이행 |
| 지각 30분 이상 | 시급 × 0.5 | 30분 단위 차감 |
| 체크리스트 전체 미이행 | ₩5,000 | 업무 태만 |
| 현금 차이 본인 과실 | 차이 금액 | 정산 오차 |

> ⚠️ 감점 정책은 관리자가 설정탭에서 활성화/비활성화 가능하도록 구현

### 8-2. 보너스 자동 반영

| 보너스 사유 | 금액 | 기준 |
|------------|:----:|------|
| 월간 근무점수 95점 이상 | ₩50,000 | 자동 계산 |
| 분기 무지각 | ₩30,000 | 3개월 연속 |
| 체크리스트 100% 달성 (월) | ₩20,000 | 모든 근무일 |

### 8-3. 급여 명세서 생성

## 9. 운영 자동화

### 9-1. 일일 자동 보고

| 시점 | 동작 |
|------|------|
| **매일 23:59** | 일일 마감 자동 집계 → daily_reports 테이블 저장 |
| **항목** | 매출합계, 예약건수, 체크리스트완료율, 알림건수, 현금잔액 |
| **관리자 조회** | 관리자탭 > 일일보고 하위탭 |

### 9-2. 교대 시간 자동 알림

```
교대 30분 전:
  → 현재 근무자에게 "교대 30분 전입니다. 인수인계를 작성해주세요" 팝업
  → 체크리스트 미완료 항목 있으면 "N건 미완료" 추가 표시
```

### 9-3. 영업 시작/종료 자동 감지

| 시점 | 동작 |
|------|------|
| **첫 출근 기록** | "영업 시작" 로그 기록, 오픈 체크리스트 타이머 시작 |
| **마지막 퇴근** | "영업 종료" 로그 기록, 일일보고 자동 생성 |

CREATE TABLE IF NOT EXISTS alert_logs ( ... );

## 현재 시스템과의 연동 포인트

| 기존 기능 | 연동 방식 |
|----------|----------|
| 출퇴근 기록 | → 지각 감지 → alert_logs 기록 → 점수 감점 |
| 체크리스트 완료 | → 시간 체크 → 미완료 알림 → 점수 반영 |
| 인수인계 작성 | → 마감 체크리스트 연동 (미작성 시 마감 불가) |
| 현금 잔액 | → 마감 시 자동 검증 → 차이 기록 |
| 스케줄 | → 무단결근 감지, 초과근무 감지, 충돌 검증 |
| 급여 계산 | → 점수/경고/감점 자동 반영 |
| MainForm 타이머 | → 기존 miceTimer 활용하여 알림 체크 추가 |

## 요약

> **"추적하지 않으면 관리할 수 없다."**

현재 CubeManager는 **데이터 기록**에는 강하지만 **문제 감지와 패턴 분석**이 부족합니다.

이 제안서의 핵심은 3가지입니다:
1. **자동 알림** — 문제를 관리자가 찾기 전에 시스템이 먼저 알린다
2. **점수화** — 주관적 평가 대신 객관적 데이터로 근무 태도를 측정한다
3. **강제성** — 중요한 업무(인수인계, 체크리스트)는 건너뛸 수 없게 만든다

1순위 5개 기능만 구현해도 **근무 관리 품질이 체감 수준으로 향상**됩니다.
