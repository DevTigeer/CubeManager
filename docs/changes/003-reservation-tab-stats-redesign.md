## 상세 변경사항

### 1. SummaryCardRow 제거

**삭제 대상:**
- `_summaryCards` 필드
- `SummaryCardRow` 생성 + `AddCard()` 4건
- `Controls.Add(_summaryCards)` 레이아웃 등록
- `UpdateSummaryCards()` 메서드 전체
- `UpdateSummaryCards()` 호출부 (PopulateMainGrid, LoadSummaryAsync)

**효과:** 예약 그리드가 ~90px 더 높아짐 (Dock.Fill이므로 자동 확대)

---

### 2. 하단 패널 3분할

기존: 좌측 60% (지출) + 우측 40% (결제 요약)
변경: 좌측 40% (지출) + 중앙 30% (통계) + 우측 30% (결제 요약)

```csharp
// 비율 배분 (Resize 이벤트)
expensePanel.Width = (int)(bottomPanel.Width * 0.40);
statsPanel.Width  = (int)(bottomPanel.Width * 0.30);
// summaryRight는 Dock.Fill로 나머지 공간
```

---

### 3. 오늘의 통계 패널 (신규 중앙 섹션)

| 항목 | 계산 방식 | 비고 |
|------|-----------|------|
| 오늘 예약 | `_reservations.Count` | 전체 (status 무관) |
| 취소 | `status == "removed"` 건수 | |
| 노쇼 | `status == "noshow"` 건수 | 신규 상태 |
| 확정 | `status == "confirmed"` 건수 | walkin 포함하지 않음 |
| 총 인원 | confirmed + walkin 의 Headcount 합계 | 취소/노쇼 제외 |
| 객단가 | 총매출 ÷ (confirmed + walkin) 건수 | 0건이면 ₩0 표시 |
| 어제 잔돈 | 전일 CashBalance.ClosingBalance | `GetCashBalanceAsync(yesterday)` |

**스타일:**
- 패널 배경: `ColorPalette.Surface`
- 헤더 "오늘의 통계": Bold 10.5pt
- 각 항목: 라벨(좌) + 값(우) 쌍
- 구분선: 총인원 위에 1px Border 색상
- 어제 잔돈: AccentOrange 색상

---

### 4. 노쇼(noshow) 상태

**현재 상태 종류:** `confirmed`, `removed`, `walkin`
**추가:** `noshow`

**우클릭 컨텍스트 메뉴 변경:**
- 확정 상태 → "취소(삭제)" / "노쇼" 표시
- 취소/노쇼 상태 → "복원" 표시
- 워크인 상태 → "취소(삭제)" / "노쇼" 표시

**노쇼 행 스타일:**
- 상태 태그: AccentOrange 배경 + AccentOrange 텍스트, "노쇼" 표시
- 행 스타일: 회색 텍스트 + 취소선 (removed와 동일 패턴)
- 결제 셀: 비활성화 (ReadOnly)

**DB 변경:** 없음 (status는 TEXT 컬럼, "noshow" 문자열 그대로 저장)
